OCLint is a static code analysis tool for improving quality and reducing defects by inspecting Objective-C/C++/C code and looking for potential problems like:

  • Possible bugs - empty if/else/try/catch/finally statements
  • Unused code - unused local variables and parameters
  • Complicated code - high cyclomatic complexity and NPath complexity
  • Code smells - long method, long parameter list, redundant code
  • …

Currently, it is a standalone tool that runs on Mac OS X 10.6+, and allows command line invocation to facilitate continuous integration and continuous inspection of the code while being developed.

OCLint is far from finished, and under continuously improvement. OCLint can be safely deloyed to development environment, and we really appreciate it for your early adoption. We warmly welcome all kinds of feedbacks.

Status

[2011-11-09] Many people asked the status of this project as there is no update recently. First of all, thank you a lot for your interests and support of this project. Now, here is a clarification: we are working very hard on a big re-design of OCLint. The changes are big, like switching libClang to libTooling, a better way of traversing AST, writing custom rules becomes easier, and more - it's hard to mention all the changes in one sentence or two. Our goal is to release the new version by the end of the year, and of course, the new code will be open source, too. We have a few people testing the new version internally, and we are improving based on the feedback. Since I receive more emails ask if there is a chance to use the new version, we now provide a very early preview version for Mac and Linux (We have extracted a small ruleset to reduce the file size). However, we couldn't provide any document or help for this preview version, please understand we are working hard on releasing it in another month.
